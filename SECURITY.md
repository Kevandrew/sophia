# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues, discussions, or pull requests.

Preferred process:

1. Use private vulnerability reporting on the hosting platform (for example, GitHub Security Advisories) when enabled.
2. If private reporting is unavailable, contact maintainers through a private channel and request an embargoed disclosure flow.

Include as much detail as possible:
- affected version/commit
- reproduction steps
- impact assessment
- suggested remediation (if known)

## Disclosure Expectations

- We will acknowledge receipt as soon as practical.
- We will triage severity and scope.
- We will coordinate a fix and publication timeline.
- We may request a reasonable disclosure embargo until a patch is available.

## Scope

This policy applies to:
- Sophia CLI source code and repository metadata handling
- documented workflows that may affect integrity/reliability

Out of scope:
- vulnerabilities in third-party dependencies that are unrelated to this project's integration choices (report upstream as appropriate)
